# Document Analyzer Skill

Provides multiple document analysis methods to help understand, improve, and optimize content with different perspectives.

## Analysis Methods

### 1. Thinker
Distills documents to their core essence, focusing on fundamental concepts and principles.
- Goal: Reach the core of things, pierce appearances
- Style: Speculative, self-consistent, letting reason flow
- Language: Searche English materials, output simplified Chinese
- Format: Markdown

### 2. Distill
Optimizes articles by removing fluff and focusing on valuable ideas.
- Removes unnecessary details
- Focuses on concepts and truly valuable parts
- Maintains core message and insights

### 3. Tags
Generates technical tags for categorization and organization.
- Provides 3-5 English tags
- Lowercase, concise (hyphen, comma)
- Concrete technical terms, avoiding generic descriptors

### 4. Researcher
Creates beginner-friendly guides for learning and understanding.
- Fascinating Beginner's Guide 2025
- Standardized technical documentation
- Language: Searche English materials, output simplified Chinese
- Format: Markdown

### 5. Extract
Focuses on deep principles and design philosophy.
- Extracts underlying design concepts
- Focuses on architectural thinking
- Reveals fundamental principles

## When to Use
- When you need to analyze documents from different perspectives
- When you want to optimize or improve existing content
- When you need to generate tags or categorize information
- When creating educational or learning materials
- When you want to understand core principles behind content

## Usage Process
1. Skill will prompt you to select analysis method
2. Provide the document/content for analysis
3. Skill applies selected analysis and returns results

## Example Interactions
- "Analyze this technical documentation using Thinker mode"
- "Distill this article to focus on key concepts"
- "Generate tags for this codebase"
- "Create a beginner's guide from this API documentation"
- "Extract the design principles from this architecture document"
